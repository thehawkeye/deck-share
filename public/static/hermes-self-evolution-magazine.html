The Evolution Pipeline

10-Step Optimization Loop

1–2SELECT + BUILD

Pick a skill, build eval dataset

Three data sources: synthetic generation, golden examples, or real session traces from ~/.hermes/sessions/*.json

3–4WRAP + CONFIGURE

Wrap as DSPy module, set up GEPA

Skill text becomes a DSPy Signature; GEPA optimizer configured with fitness metric

5EVOLVE

GEPA optimization — 10 iterations

Reads execution traces to understand WHY things fail; proposes targeted mutations; evaluates against metric

6–9VALIDATE + REPORT

Constraint gates + holdout evaluation

Test suite, size limits, caching compat, semantic preservation, PR review — all must pass

10DEPLOY

Save evolved skill + metrics.json

Before/after scores, size delta, constraints passed. Human reviews before merge to hermes-agent repo.

Risk Assessment

Guardrails & Mitigations

Evolved skill performs worse
Risk: Low. Constraint gates catch regressions. Holdout evaluation compares before/after scores. Human reviews evolved output before merge.
API cost overrun
Risk: Low. Hard-capped iterations (default 10). ~$2–10 per run. Occasional use only. Well within ₹2k/month target.
Dependency conflicts
Risk: Low. Isolated Python venv. pip install -e "[dev]" in dedicated directory. No impact on hermes-agent venv.
Semantic drift in critical skill
Risk: Medium. Gate #4 enforces semantic preservation. Always review critical skills manually. Start with non-critical skills first.
Session data leak into training
Risk: Very Low. Built-in secret detection filters API keys, tokens, and PII from session data before it enters training.
Darwinian Evolver (AGPL v3)
Risk: None. Only Phase 4 (code evolution, not yet implemented). Separate external CLI. Not needed for Phase 1 skill evolution.

5 Guardrails Every Variant Must Pass

  • 1 Full test suite — pytest tests/ -q must pass 100%
  • 2 Size limits — Skills ≤15KB, tool descriptions ≤500 chars
  • 3 Caching compatibility — No mid-conversation changes
  • 4 Semantic preservation — Must not drift from original purpose
  • 5 PR review — All changes go through human review, never direct commit
