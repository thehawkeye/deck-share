System Architecture

Two Engines, One Loop

🟡 Engine A: ME-SE (Reactive)

Location: ~/.hermes/scripts/self_evolution/

Schedule: Hourly collect → 6h review → 2× daily orchestrate

Function: Detects errors in logs → Ranks by impact × feasibility → Generates fix goals → Verifies fixes

🟢 Engine B: NR-HSE (Proactive)

Location: ~/Documents/Codex/hermes-agent-self-evolution/

Schedule: On-demand / triggered by ME-SE goals

Function: Reads SKILL.md → Generates eval dataset → Evolves via DSPy+GEPA → Gates through constraints → Outputs improved skill

"ME-SE catches problems after they occur. NR-HSE evolves skills to prevent problems at the source. Together, they create a genuinely self-improving agent."

1
COLLECT

Operational Cadence

Weekly Optimization Schedule

M
MONDAY 8AM

Weekly skill optimizer runs. Reads last week's error report. Identifies top-3 error-prone skills. Evolves each via NR-HSE.

H
HOURLY

ME-SE error collector scans all logs. Collects anomalies into errors_collected.jsonl.

6H
EVERY 6 HOURS

ME-SE review prioritizer ranks errors. Generates fix goals for top issues.

2×
TWICE DAILY

ME-SE goal orchestrator executes goals. Routes to evolve_skill or manual fix.

PR
ON EVOLUTION

Evolved skills posted to #admin. Human reviews diff + metrics. Approves or rejects.
